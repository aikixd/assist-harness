See [docs/guide.md](/home/aikixd/Dev/Personal/assist-harness/docs/guide.md) for repo conventions.

## Collaboration Rules

- If the user asks a question, answer the question rather than implementing changes.
- Before making edits or implementing code, align with the user on the action plan and approach.
- Planning artifacts are allowed before implementation, including sketches, plans, specs, and doc drafts.
- If a request or plan is vague in a way that could lead to meaningfully different results, raise the ambiguity and ask for clarification before proceeding.
- If the user asks to create a symbol and the requested name appears to contain a typo, ask whether that name was intended before creating it.
